# Автоматический деплой HTTPS

## Быстрый способ (одна команда)

Просто выполните в терминале из корня проекта:

```bash
bash deploy_https.sh
```

Скрипт автоматически:
1. ✅ Установит sshpass (если нужно)
2. ✅ Загрузит скрипт настройки на сервер
3. ✅ Выполнит все команды на сервере
4. ✅ Настроит HTTPS для milviar.ru
5. ✅ Получит SSL сертификат от Let's Encrypt
6. ✅ Настроит WebSocket через WSS

**Время выполнения:** 2-3 минуты

---

## Что нужно перед запуском

### 1. Настройте DNS запись

В панели управления доменом `milviar.ru`:
- Добавьте A-запись: `@` → `83.166.246.225`
- Подождите 10-15 минут для распространения

### 2. Проверьте DNS

```bash
dig milviar.ru +short
# Должно показать: 83.166.246.225
```

### 3. Запустите деплой

```bash
bash deploy_https.sh
```

---

## Альтернативный способ (если deploy_https.sh не работает)

### Вариант 1: Ручная загрузка скрипта

```bash
# Загрузите скрипт на сервер
scp auto_setup_https.sh root@83.166.246.225:/root/

# Подключитесь к серверу
ssh root@83.166.246.225

# Выполните скрипт
bash /root/auto_setup_https.sh
```

### Вариант 2: Прямое выполнение команд

Если скрипт не работает, используйте команды из `COMMANDS.txt` - просто копируйте и вставляйте по очереди.

## Решение проблем

### Ошибка: sshpass не установлен

**macOS:**
```bash
brew install hudochenkov/sshpass/sshpass
```

**Linux:**
```bash
sudo apt-get install sshpass
```

### Ошибка: DNS не распространился

Подождите еще 15-30 минут и проверьте:
```bash
dig milviar.ru +short
```

### Ошибка: Certbot не может получить сертификат

Проверьте логи на сервере:
```bash
ssh root@83.166.246.225
journalctl -u certbot -n 50
```

Убедитесь, что:
- DNS запись правильная
- Порт 80 открыт
- Nginx работает
