# Инструкция для заказчика: Решение проблемы с мобильным интернетом

## Быстрая проверка

### Шаг 1: Проверка DNS

**На iPhone заказчика:**

1. Откройте **Safari**
2. В адресной строке введите: `https://milviar.ru`
3. Нажмите Enter

**Ожидаемый результат:**
- ✅ Должно открыться БЕЗ предупреждений о безопасности
- ✅ Должен показаться какой-то ответ (даже если ошибка)

**Если не открывается:**
- ❌ Проблема с DNS или блокировкой оператора
- Попробуйте через Wi-Fi (должно работать)
- Попробуйте через VPN

### Шаг 2: Проверка версии приложения

**В TestFlight:**
1. Откройте TestFlight
2. Найдите ваше приложение
3. Проверьте версию - должна быть **1.0.0+16** или выше

**Если версия старая:**
- Удалите приложение
- Установите заново из TestFlight
- Убедитесь, что установлена последняя версия

### Шаг 3: Проверка интернета

**На iPhone:**
1. Откройте **Настройки** → **Сотовая связь**
2. Убедитесь, что **Сотовые данные** включены
3. Попробуйте открыть любой сайт в Safari через мобильный интернет

**Если интернет не работает вообще:**
- Проблема с оператором/тарифом
- Обратитесь к оператору

### Шаг 4: Перезагрузка устройства

**Простое решение, которое часто помогает:**
1. Зажмите кнопку питания + кнопку уменьшения громкости
2. Выберите **"Перезагрузить"**
3. Подождите перезагрузки
4. Попробуйте снова

### Шаг 5: Очистка кеша приложения

**Если приложение уже установлено:**
1. Удалите приложение с устройства
2. Перезагрузите iPhone
3. Установите приложение заново из TestFlight
4. Попробуйте войти

## Детальная диагностика

### Что проверить у заказчика:

1. **Версия iOS:**
   - Настройки → Основные → Об этом устройстве
   - Должна быть iOS 14.0 или выше

2. **Оператор:**
   - Какой оператор использует заказчик?
   - Мегафон, МТС, Билайн, Теле2?

3. **VPN:**
   - Включен ли VPN на устройстве?
   - Настройки → VPN
   - Если включен - отключите и попробуйте снова

4. **Корпоративный профиль:**
   - Есть ли корпоративный профиль на устройстве?
   - Настройки → Основные → Профили и управление устройством
   - Корпоративные профили могут блокировать трафик

5. **Ограничения:**
   - Настройки → Экранное время → Ограничения контента
   - Убедитесь, что установка приложений разрешена

## Возможные причины проблемы

### 1. DNS не распространился на устройстве заказчика

**Решение:**
- Подождите 24 часа
- Или используйте другой DNS:
  - Настройки → Wi-Fi → (i) рядом с сетью → DNS → Настроить DNS → Вручную
  - Добавьте: `8.8.8.8` и `8.8.4.4` (Google DNS)

### 2. Оператор блокирует домен

**Симптомы:**
- Wi-Fi работает, мобильный интернет не работает
- Safari не открывает https://milviar.ru через мобильные данные

**Решение:**
- Используйте VPN
- Или используйте Wi-Fi

### 3. Старая версия приложения

**Симптомы:**
- Приложение использует старый URL (http://83.166.246.225)

**Решение:**
- Удалите приложение
- Установите новую версию из TestFlight (1.0.0+16)

### 4. Проблемы с сертификатом

**Симптомы:**
- Предупреждение о безопасности в Safari
- Ошибки SSL/TLS

**Решение:**
- Проверьте дату и время на устройстве (должны быть правильными)
- Попробуйте через Wi-Fi

## Сбор логов для диагностики

Попросите заказчика:

1. **Включить логирование** (если есть опция в приложении)
2. **Попробовать войти через мобильный интернет**
3. **Скопировать все сообщения из консоли** (если доступны)
4. **Отправить вам**

**Что особенно важно в логах:**
- URL запросов (должны быть `https://milviar.ru`)
- Тип сети (Cellular vs WiFi)
- Ошибки подключения
- Статус ответов сервера

## Альтернативные решения

### Вариант 1: Использовать VPN

Если оператор блокирует домен:
1. Установите VPN приложение (например, Cloudflare WARP - бесплатный)
2. Включите VPN
3. Попробуйте войти в приложение

### Вариант 2: Использовать Wi-Fi

Если мобильный интернет не работает:
- Используйте Wi-Fi для работы приложения
- Это временное решение, пока не решим проблему с мобильными данными

### Вариант 3: Сменить DNS

На iPhone:
1. Настройки → Wi-Fi
2. Нажмите (i) рядом с сетью Wi-Fi
3. Прокрутите вниз до "DNS"
4. Нажмите "Настроить DNS"
5. Выберите "Вручную"
6. Добавьте: `8.8.8.8` и `8.8.4.4`
7. Сохраните
8. Попробуйте снова

## Проверка работы сервера

**Для вас (разработчика):**

Проверьте, что сервер работает:
```bash
curl -I https://milviar.ru
# Должно показать HTTP/2 200 или 301
```

Проверьте SSL сертификат:
```bash
openssl s_client -connect milviar.ru:443 -servername milviar.ru
# Должен показать валидный сертификат от Let's Encrypt
```

---

## Если ничего не помогает

1. **Попросите заказчика попробовать через Wi-Fi** - если работает, значит проблема в мобильном интернете
2. **Попросите заказчика попробовать через VPN** - если работает, значит оператор блокирует
3. **Проверьте логи на сервере** - возможно, запросы вообще не доходят до сервера
4. **Проверьте логи приложения** - там должна быть информация об ошибках
