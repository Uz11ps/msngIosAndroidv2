# КРИТИЧЕСКИ ВАЖНО: Исправление разрешений iOS

## Главная проблема
iOS не видит разрешения из `Info.plist`, потому что они не правильно интегрированы в проект Xcode.

## Решение (выполните ВСЕ шаги по порядку)

### ⚠️ ВАЖНО: Вы уже в директории проекта!
Не выполняйте `cd msngIosAndroidv2` - вы уже там!

### Шаг 1: Правильная очистка (выполните в терминале на Mac)

```bash
# 1. Очистка Flutter
flutter clean

# 2. Получение зависимостей (ВАЖНО: сначала это!)
flutter pub get

# 3. Очистка iOS зависимостей
cd ios
rm -rf Pods Podfile.lock
pod install
cd ..

# 4. Очистка кэша Xcode
rm -rf ~/Library/Developer/Xcode/DerivedData/*
```

### Шаг 2: Запустите эмулятор ПЕРЕД сбросом разрешений

```bash
# Откройте эмулятор
open -a Simulator

# ПОДОЖДИТЕ, пока эмулятор полностью загрузится (может занять 30-60 секунд)

# Затем удалите приложение (если установлено)
xcrun simctl uninstall booted com.vvedenskii.messenger

# Сбросьте настройки приватности
xcrun simctl privacy booted reset all
```

### Шаг 3: КРИТИЧЕСКИ ВАЖНО - Проверка в Xcode

**Это самый важный шаг!** Без этого разрешения не будут работать!

1. **Откройте проект в Xcode:**
   ```bash
   open ios/Runner.xcworkspace
   ```
   ⚠️ НЕ открывайте `.xcodeproj`, только `.xcworkspace`!

2. **В Xcode выполните следующие проверки:**

   **A. Проверка таргета:**
   - В левой панели найдите синий значок проекта `Runner`
   - Кликните на него
   - В центральной панели выберите таргет **`Runner`** (НЕ RunnerTests!)
   - Убедитесь, что выбран правильный таргет

   **B. Проверка Info.plist через вкладку Info:**
   - Выберите таргет `Runner`
   - Перейдите на вкладку **"Info"** (вверху)
   - Найдите раздел **"Custom iOS Target Properties"**
   - **Проверьте наличие следующих ключей:**
     - `Privacy - Microphone Usage Description`
     - `Privacy - Camera Usage Description`
     - `Privacy - Photo Library Usage Description`
     - `Privacy - Photo Library Additions Usage Description`
   
   **C. Если ключей НЕТ в Xcode (но есть в файле Info.plist):**
   - Это означает, что Xcode не видит файл!
   - Нажмите кнопку **"+"** внизу списка свойств
   - Добавьте каждый ключ вручную:
     - Выберите из выпадающего списка или введите точное имя ключа
     - Введите значение из файла `Info.plist`
   
   **D. Если ключи ЕСТЬ, но значения пустые или неправильные:**
   - Кликните на каждое значение
   - Введите правильный текст из `Info.plist`

   **E. Проверка Build Settings:**
   - Оставаясь на таргете `Runner`
   - Перейдите на вкладку **"Build Settings"**
   - В поиске введите: `INFOPLIST_FILE`
   - Найдите строку `INFOPLIST_FILE`
   - Убедитесь, что значение: **`Runner/Info.plist`**
   - Если значение другое, измените его
   
   **F. Проверка GENERATE_INFOPLIST_FILE:**
   - В том же разделе Build Settings
   - В поиске введите: `GENERATE_INFOPLIST`
   - Найдите `GENERATE_INFOPLIST_FILE`
   - Для таргета `Runner` значение должно быть: **`NO`** или отсутствовать
   - Если значение `YES`, измените на `NO`

3. **Сохраните проект:**
   - Нажмите `Cmd + S` (или File → Save)
   - Закройте Xcode

## Если разрешения все еще не работают

### Альтернативное решение: Добавление через Xcode UI

Если файл `Info.plist` не синхронизируется с Xcode:

1. Откройте `ios/Runner.xcworkspace` в Xcode
2. Выберите проект → Runner target → **Info tab**
3. Удалите все существующие ключи приватности (если есть)
4. Нажмите **"+"** и добавьте каждый ключ заново:

   **Ключ 1:**
   - Key: `Privacy - Microphone Usage Description`
   - Type: `String`
   - Value: `Приложению необходим доступ к микрофону для записи голосовых сообщений и аудиозвонков`

   **Ключ 2:**
   - Key: `Privacy - Camera Usage Description`
   - Type: `String`
   - Value: `Приложению необходим доступ к камере для отправки фотографий и видеозвонков`

   **Ключ 3:**
   - Key: `Privacy - Photo Library Usage Description`
   - Type: `String`
   - Value: `Приложению необходим доступ к галерее для выбора фотографий`

   **Ключ 4:**
   - Key: `Privacy - Photo Library Additions Usage Description`
   - Type: `String`
   - Value: `Приложению необходим доступ для сохранения фотографий в галерею`

5. Сохраните проект (`Cmd + S`)
6. Выполните полную пересборку (Шаг 4)
7. Удалите приложение с эмулятора и переустановите

## Проверка результата

После выполнения всех шагов выполните:

```bash
# Проверьте, что Info.plist содержит правильные ключи
cat ios/Runner/Info.plist | grep -A 1 "NSMicrophoneUsageDescription"
cat ios/Runner/Info.plist | grep -A 1 "NSCameraUsageDescription"
cat ios/Runner/Info.plist | grep -A 1 "NSPhotoLibraryUsageDescription"
```

Все команды должны вывести соответствующие ключи и значения.

## Почему это важно?

iOS читает разрешения из `Info.plist` только если:
1. Файл правильно указан в `INFOPLIST_FILE` в Build Settings
2. `GENERATE_INFOPLIST_FILE` отключен (или отсутствует)
3. Ключи присутствуют в файле, который Xcode видит в своем UI
4. Приложение пересобрано после изменений

Если хотя бы одно условие не выполнено, iOS не покажет запросы разрешений и они не появятся в настройках.
