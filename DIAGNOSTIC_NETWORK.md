# Диагностика проблем с мобильным интернетом

## Проверка для заказчика

Попросите заказчика выполнить следующие проверки:

### 1. Проверка DNS

**На устройстве заказчика:**
- Откройте Safari
- Введите в адресной строке: `https://milviar.ru`
- Должно открыться БЕЗ предупреждений о безопасности

**Если не открывается:**
- Проверьте, что DNS запись для `milviar.ru` правильная
- Попробуйте через Wi-Fi (должно работать)
- Попробуйте через другую мобильную сеть

### 2. Проверка версии приложения

**В приложении:**
- Откройте настройки приложения (если есть)
- Проверьте версию - должна быть **1.0.0+15** или выше

**В TestFlight:**
- Убедитесь, что установлена последняя версия
- Если есть обновление - обновите приложение

### 3. Проверка логов

**Попросите заказчика:**
1. Откройте приложение
2. Попробуйте войти
3. Скопируйте все сообщения из консоли (если доступны)
4. Отправьте вам

**Что искать в логах:**
- `📱 Network type: Cellular/Mobile data` - используется мобильный интернет
- `🔐 Attempting login` - попытка входа
- `📡 Response status` - статус ответа сервера
- Ошибки типа `Failed host lookup`, `Connection refused`, `SSL`, `TLS`

### 4. Проверка интернет-соединения

**На устройстве заказчика:**
- Откройте Safari
- Попробуйте открыть любой сайт (например, google.com)
- Если не открывается - проблема с интернетом в целом

### 5. Проверка VPN и прокси

**Убедитесь, что:**
- VPN отключен (если был включен)
- Прокси не настроен в настройках Wi-Fi/мобильных данных
- Корпоративный профиль не блокирует трафик

### 6. Переустановка приложения

**Если ничего не помогает:**
1. Удалите приложение с устройства
2. Перезагрузите устройство
3. Установите приложение заново из TestFlight
4. Попробуйте войти

## Возможные проблемы и решения

### Проблема 1: DNS не распространился

**Симптомы:**
- Приложение не может подключиться
- Safari не открывает https://milviar.ru

**Решение:**
- Проверьте DNS запись в панели управления доменом
- Подождите 24 часа для полного распространения DNS
- Попробуйте использовать другой DNS (например, Google DNS: 8.8.8.8)

### Проблема 2: Оператор блокирует домен

**Симптомы:**
- Wi-Fi работает, мобильный интернет не работает
- Safari не открывает https://milviar.ru через мобильные данные

**Решение:**
- Используйте VPN
- Или используйте Wi-Fi

### Проблема 3: Проблемы с сертификатом

**Симптомы:**
- Предупреждение о безопасности в Safari
- Ошибки SSL/TLS в логах

**Решение:**
- Проверьте, что сертификат валидный: `openssl s_client -connect milviar.ru:443`
- Убедитесь, что сертификат не истек

### Проблема 4: Проблемы с WebSocket

**Симптомы:**
- Вход работает, но сообщения не отправляются
- Ошибки подключения Socket

**Решение:**
- Проверьте логи Socket.IO
- Убедитесь, что WSS работает: `wscat -c wss://milviar.ru`

## Сбор логов для диагностики

Попросите заказчика:

1. **Включить логирование в приложении** (если есть такая опция)
2. **Попробовать войти через мобильный интернет**
3. **Скопировать все сообщения из консоли**
4. **Отправить вам**

**Что особенно важно:**
- URL запросов (должны быть `https://milviar.ru`)
- Тип сети (Cellular vs WiFi)
- Ошибки подключения
- Статус ответов сервера

---

## Альтернативное решение

Если проблема сохраняется, можно:

1. **Использовать VPN** - обойдет любые блокировки оператора
2. **Использовать Wi-Fi** - всегда работает
3. **Проверить настройки оператора** - некоторые операторы требуют специальных настроек
