# 🔧 Исправление краша на эмуляторе iOS

## Проблема:
Приложение крашится при запуске на эмуляторе с ошибкой `SIGTERM` и не может установиться.

## Причины:
1. **Конфликт Git** - локальные изменения в iOS файлах конфликтуют с удаленными
2. **Эмулятор в состоянии Shutdown** - эмулятор не готов к установке приложения
3. **Несинхронизированные iOS файлы** - конфликтующие настройки проекта

---

## ✅ Решение:

### Шаг 1: Разрешите конфликт Git

```bash
# Сохраните локальные изменения (если они нужны)
git stash

# Или закоммитьте их
git add ios/
git commit -m "Save local iOS changes"

# Затем обновите проект
git pull
```

### Шаг 2: Перезапустите эмулятор

```bash
# Закройте все эмуляторы
killall Simulator

# Или через Xcode:
# Xcode → Window → Devices and Simulators → выберите эмулятор → Delete

# Запустите эмулятор заново
open -a Simulator
```

### Шаг 3: Очистите проект

```bash
cd msngIosAndroidv2

# Очистите Flutter
flutter clean

# Очистите iOS зависимости
cd ios
rm -rf Pods Podfile.lock
pod install
cd ..

# Очистите Xcode Derived Data
rm -rf ~/Library/Developer/Xcode/DerivedData/*
```

### Шаг 4: Пересоберите и запустите

```bash
flutter pub get
flutter run
```

---

## 🔍 Если проблема сохраняется:

### Вариант 1: Используйте другой эмулятор

```bash
# Список доступных эмуляторов
xcrun simctl list devices

# Запустите другой эмулятор
flutter run -d <device-id>
```

### Вариант 2: Пересоздайте эмулятор

1. Xcode → Window → Devices and Simulators
2. Выберите проблемный эмулятор
3. Нажмите "Delete"
4. Создайте новый эмулятор
5. Запустите приложение

### Вариант 3: Проверьте логи Xcode

```bash
# Откройте Xcode
open ios/Runner.xcworkspace

# Запустите через Xcode (Product → Run)
# Проверьте логи в консоли Xcode
```

## 📱 Альтернатива: Тестирование на реальном устройстве

Если эмулятор продолжает крашиться, протестируйте на реальном iPhone:

1. Подключите iPhone к Mac
2. Доверьтесь компьютеру на iPhone
3. Запустите:
```bash
flutter run -d <your-iphone-id>
```
