## Как использовать:

### Для заказчика:

1. **Если авторизация не работает:**
   - На экране входа нажмите кнопку **"Диагностика подключения"** (внизу экрана)
   - Дождитесь завершения проверки
   - Сделайте скриншот результатов
   - Отправьте разработчику

2. **Если появилась ошибка:**
   - Прочитайте сообщение об ошибке
   - Нажмите кнопку **"Диагностика"** в сообщении
   - Следуйте инструкциям на экране

### Для разработчика:

1. **Попросите заказчика:**
   - Открыть диагностический экран
   - Сделать скриншот результатов
   - Попробовать с мобильного интернета (если использует WiFi)
   - Попробовать переустановить приложение

2. **Проверьте сервер:**
   ```bash
   # Проверка доступности
   curl -X POST http://83.166.246.225:3000/api/auth/email-register \
     -H "Content-Type: application/json" \
     -d '{}'
   
   # Проверка логов
   pm2 logs messenger-backend
   ```

## Возможные проблемы и решения:

### Проблема: "Ошибка сети"
**Решение:**
- Проверить интернет-соединение
- Попробовать другую сеть (мобильный интернет вместо WiFi)
- Проверить, не блокирует ли провайдер доступ

### Проблема: "Сервер вернул HTML вместо JSON"
**Решение:**
- Сервер может быть недоступен
- Проверить логи сервера
- Проверить доступность сервера через браузер

### Проблема: "Превышено время ожидания"
**Решение:**
- Медленное интернет-соединение
- Попробовать другую сеть
- Проверить скорость интернета

### Проблема: "Сервер вернул неверный формат данных"
**Решение:**
- Возможно, сервер перегружен
- Проверить логи сервера
- Перезапустить сервер
