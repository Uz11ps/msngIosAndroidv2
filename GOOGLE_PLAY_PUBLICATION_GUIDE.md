## 1. Подготовка приложения

### 1.1. Проверка конфигурации

Убедитесь, что в `android/app/build.gradle` правильно настроены:

```gradle
android {
    namespace = "com.messenger.messenger_app"
    compileSdk = 35
    
    defaultConfig {
        applicationId = "com.messenger.messenger_app"  // Уникальный ID приложения
        minSdk = 23  // Минимальная версия Android
        targetSdk = flutter.targetSdkVersion
        versionCode = 1  // Увеличивайте при каждом обновлении
        versionName = "1.0.0"  // Версия для пользователей
    }
}
```

### 1.2. Обновление версии

В `pubspec.yaml`:
```yaml
version: 1.0.0+1
# Формат: versionName+versionCode
# 1.0.0 - версия для пользователей
# +1 - версия для системы (увеличивайте при каждом обновлении)
```

### 1.3. Проверка AndroidManifest.xml

Убедитесь, что в `android/app/src/main/AndroidManifest.xml` есть:

```xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android">
    <uses-permission android:name="android.permission.INTERNET"/>
    <uses-permission android:name="android.permission.RECORD_AUDIO"/>
    <uses-permission android:name="android.permission.CAMERA"/>
    <uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>
    <!-- Другие необходимые разрешения -->
    
    <application
        android:label="Messenger App"
        android:name="${applicationName}"
        android:icon="@mipmap/ic_launcher">
        <!-- ... -->
    </application>
</manifest>
```

## 2. Создание аккаунта разработчика

### 2.1. Регистрация в Google Play Console

1. Перейдите на https://play.google.com/console
2. Войдите с вашим Google аккаунтом
3. Нажмите "Создать аккаунт разработчика"
4. Заполните форму:
   - **Имя разработчика**: Ваше имя или название компании
   - **Email**: Ваш email
   - **Телефон**: Номер телефона для связи
   - **Страна/регион**: Ваша страна

### 2.2. Оплата регистрационного взноса

- **Стоимость**: $25 (единоразово, навсегда)
- **Способы оплаты**: Кредитная/дебетовая карта, PayPal
- После оплаты аккаунт активируется в течение нескольких часов

### 2.3. Принятие соглашения разработчика

Прочитайте и примите соглашение разработчика Google Play.

## 3. Подготовка материалов

### 3.1. Иконка приложения

**Требования:**
- Формат: PNG (без прозрачности)
- Размер: 512x512 пикселей
- Форма: Квадрат
- Фон: Непрозрачный

**Создание:**
- Используйте графический редактор (Photoshop, Figma, Canva)
- Или создайте через онлайн-генераторы иконок

**Размещение:**
- Сохраните как `android/app/src/main/res/mipmap-xxxhdpi/ic_launcher.png`
- Или используйте `flutter_launcher_icons` пакет

### 3.2. Скриншоты

**Требования:**
- Минимум: 2 скриншота
- Рекомендуется: 4-8 скриншотов
- Размеры:
  - Телефон: минимум 320px, максимум 3840px (ширина или высота)
  - Планшет (7"): минимум 320px, максимум 3840px
  - Планшет (10"): минимум 320px, максимум 3840px
- Формат: PNG или JPEG
- Соотношение сторон: 16:9 или 9:16

**Что показать:**
- Главный экран
- Экран чатов
- Экран сообщений
- Профиль пользователя
- Групповые чаты
- Звонки (если есть)

**Как сделать:**
1. Запустите приложение на эмуляторе или устройстве
2. Сделайте скриншоты (Power + Volume Down на Android)
3. Или используйте инструменты для создания скриншотов

### 3.3. Графические материалы

**Feature Graphic (Графический баннер):**
- Размер: 1024 x 500 пикселей
- Формат: PNG или JPEG
- Используется в Google Play Store

**Промо-графика (опционально):**
- Размер: 180 x 120 пикселей
- Для рекламных кампаний

### 3.4. Текстовые материалы

**Название приложения:**
- Максимум: 50 символов

- Используйте форматирование (переносы строк, списки)
- Опишите основные функции

**Пример описания:**

## 4. Настройка подписи приложения

Google Play требует, чтобы приложение было подписано цифровой подписью.

### 4.1. Создание ключа подписи

**ВНИМАНИЕ:** Сохраните ключ в безопасном месте! Без него вы не сможете обновлять приложение!

```bash
# Перейдите в директорию android/app
cd android/app

# Создайте ключ (замените данные на свои)
keytool -genkey -v -keystore upload-keystore.jks -keyalg RSA -keysize 2048 -validity 10000 -alias upload

# Введите данные:
# - Пароль для хранилища ключей (запомните его!)
# - Имя и фамилию
# - Название организации
# - Город, регион, страна
```

### 4.2. Создание файла key.properties

Создайте файл `android/key.properties`:

```properties
storePassword=ВАШ_ПАРОЛЬ_ХРАНИЛИЩА
keyPassword=ВАШ_ПАРОЛЬ_КЛЮЧА
keyAlias=upload
storeFile=upload-keystore.jks
```

**ВАЖНО:** Добавьте `key.properties` в `.gitignore`!

### 4.3. Настройка build.gradle

Обновите `android/app/build.gradle`:

```gradle
def keystoreProperties = new Properties()
def keystorePropertiesFile = rootProject.file('key.properties')
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
}

android {
    // ... существующий код ...

    signingConfigs {
        release {
            keyAlias keystoreProperties['keyAlias']
            keyPassword keystoreProperties['keyPassword']
            storeFile keystoreProperties['storeFile'] ? file(keystoreProperties['storeFile']) : null
            storePassword keystoreProperties['storePassword']
        }
    }

    buildTypes {
        release {
            signingConfig signingConfigs.release
            // Отключите минификацию если нужно
            // minifyEnabled false
            // shrinkResources false
        }
    }
}
```

### 4.4. Перемещение ключа

Переместите `upload-keystore.jks` в `android/`:

```bash
mv android/app/upload-keystore.jks android/
```

Обновите путь в `key.properties`:
```properties
storeFile=../upload-keystore.jks
```

## 5. Сборка App Bundle

Google Play требует формат AAB (Android App Bundle), а не APK.

### 5.1. Сборка

```bash
# Вернитесь в корень проекта
cd ../..

# Очистите предыдущие сборки
flutter clean

# Получите зависимости
flutter pub get

# Соберите App Bundle
flutter build appbundle --release
```

### 5.2. Результат

Файл будет создан в:
```
build/app/outputs/bundle/release/app-release.aab
```

**Размер:** Обычно меньше APK, так как Google Play создаст оптимизированные APK для каждого устройства.

   - **Тип приложения**: Приложение

### 6.2. Настройка аккаунта разработчика (если нужно)

Если это первый раз:
- Заполните контактную информацию

**Проверки Google Play:**
- Размер приложения

## 10. После публикации

### 10.1. Мониторинг

**Google Play Console предоставляет:**
- Статистику установок
- Отзывы пользователей
- Рейтинги
- Данные о сбоях
- Аналитику доходов (если есть)

### 10.2. Обновления приложения

**Для выпуска обновления:**

1. Увеличьте версию в `pubspec.yaml`:
   ```yaml
   version: 1.0.1+2  # +2 - новый versionCode
   ```

2. Соберите новый App Bundle:
   ```bash
   flutter build appbundle --release
   ```

3. Загрузите в Google Play Console:
   - "Выпуск" → "Production" → "Создать выпуск"
   - Загрузите новый AAB
   - Заполните "Что нового"
   - Отправьте на проверку

### 10.3. Ответы на отзывы

- Регулярно проверяйте отзывы
- Отвечайте на вопросы пользователей
- Исправляйте проблемы, о которых пишут

### 10.4. Продвижение

**Способы продвижения:**
- Социальные сети
- Реклама в Google Ads
- Партнерства с блогерами
- Оптимизация описания для поиска (ASO)

## Часто задаваемые вопросы

### Q: Сколько стоит публикация?
**A:** $25 единоразово за аккаунт разработчика. Обновления бесплатны.

### Q: Сколько времени занимает проверка?
**A:** Обычно 1-3 дня, может быть до 7 дней.

### Q: Можно ли изменить название приложения после публикации?
**A:** Да, но это может повлиять на поиск и узнаваемость.

### Q: Нужен ли мне веб-сайт?
**A:** Нет, но нужна страница с политикой конфиденциальности (можно на GitHub Pages).

### Q: Что делать, если приложение отклонили?
**A:** Google Play отправит email с причиной. Исправьте проблемы и отправьте снова.

### Q: Можно ли удалить приложение?
**A:** Да, можно удалить из Google Play, но пользователи, которые уже установили, сохранят его.
