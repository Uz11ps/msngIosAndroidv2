# 🔐 Как получить разрешения на iOS

## Проблема:
Системный диалог iOS для запроса разрешений не появляется, и разрешения нельзя включить в настройках.

---

## ✅ Решение:

### Шаг 1: Удалите приложение с эмулятора/устройства

**На эмуляторе:**
- Долгим нажатием на иконку приложения → Удалить приложение
- Или через Xcode: Window → Devices and Simulators → выберите устройство → удалите приложение

**На реальном устройстве:**
- Долгим нажатием на иконку → Удалить приложение

### Шаг 2: Сбросьте разрешения эмулятора (если используете эмулятор)

```bash
# Сбросьте все разрешения эмулятора
xcrun simctl privacy booted reset all

# Или сбросьте только для вашего приложения (замените BUNDLE_ID)
xcrun simctl privacy booted reset all com.vvedenskii.messenger
```

### Шаг 3: Пересоберите и переустановите приложение

```bash
cd msngIosAndroidv2

# Очистите проект
flutter clean

# Получите зависимости
flutter pub get

# Пересоберите и запустите
flutter run
```

### Шаг 4: Проверьте логи

При первом использовании функции (звонок, запись голосового сообщения) в консоли должны появиться логи:
```
🎤 Microphone permission status: PermissionStatus.denied
🎤 Requesting microphone permission...
🎤 Microphone permission after request: PermissionStatus.granted
```

## 🔍 Если системный диалог все еще не появляется:

### Вариант 1: Проверьте Info.plist в Xcode

1. Откройте проект в Xcode:
```bash
cd ios
open Runner.xcworkspace
```

2. Выберите `Runner` в навигаторе слева
3. Выберите вкладку `Info`
4. Убедитесь, что есть следующие ключи:
   - ✅ `Privacy - Microphone Usage Description` (NSMicrophoneUsageDescription)
   - ✅ `Privacy - Camera Usage Description` (NSCameraUsageDescription)
   - ✅ `Privacy - Photo Library Usage Description` (NSPhotoLibraryUsageDescription)

5. Если ключей нет - добавьте их вручную через Xcode

### Вариант 2: Проверьте, что Info.plist включен в сборку

1. В Xcode выберите `Runner` → `Build Phases`
2. Найдите `Copy Bundle Resources`
3. Убедитесь, что `Info.plist` там есть
4. Если нет - добавьте его

### Вариант 3: Включите разрешения вручную через настройки эмулятора

1. Откройте **Настройки** на эмуляторе
2. Найдите **Messenger App** в списке приложений
3. Если приложения нет - значит разрешение еще не запрашивалось
4. Если есть - включите переключатели для микрофона и камеры

### Вариант 4: Используйте реальное устройство

На реальном iPhone разрешения работают надежнее:
1. Подключите iPhone к Mac
2. Доверьтесь компьютеру на iPhone
3. Запустите:
```bash
flutter run -d <your-iphone-id>
```

---

## 📱 Альтернативный способ: Включите разрешения через настройки iOS

Если системный диалог не появляется, но приложение уже установлено:

1. Откройте **Настройки** на iPhone/эмуляторе
2. Прокрутите вниз до **Messenger App**
3. Нажмите на приложение
4. Включите переключатели:
   - ✅ Микрофон
   - ✅ Камера
   - ✅ Фото

---

## 🐛 Отладка: Проверьте логи

Добавьте больше логирования в код, чтобы понять, что происходит:

В консоли должны быть сообщения:
```
🎤 Microphone permission status: PermissionStatus.denied
🎤 Requesting microphone permission...
🎤 Microphone permission after request: PermissionStatus.granted (или denied)
```

Если статус всегда `denied` и не меняется после запроса - значит системный диалог не появляется.

## ✅ После получения разрешений:

После того, как разрешения будут предоставлены:
- Звонки будут работать
- Запись голосовых сообщений будет работать
- Выбор изображений будет работать

---

## 💡 Важно для эмулятора:

На некоторых эмуляторах iOS микрофон и камера могут не работать физически. В этом случае:
- Разрешения можно включить в настройках, но функции все равно не будут работать
- Рекомендуется тестировать на реальном устройстве
