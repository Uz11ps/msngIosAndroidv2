# Переход на HTTPS

## Что было сделано

Приложение переведено с HTTP на HTTPS для решения проблемы с блокировкой HTTP трафика операторами мобильной связи (например, Мегафон).

## Изменения

### 1. API конфигурация
- **Файл:** `lib/config/api_config.dart`
- **Изменение:** URL изменены с `http://` на `https://`
  - `baseUrl`: `https://83.166.246.225:3000`
  - `wsUrl`: `https://83.166.246.225:3000` (для WebSocket)

### 2. Настройки iOS (ATS)
- **Файл:** `ios/Runner/Info.plist`
- **Изменения:**
  - Убраны настройки для HTTP (`NSExceptionAllowsInsecureHTTPLoads: false`)
  - Оставлены настройки для HTTPS с поддержкой TLS 1.0+
  - Добавлена поддержка локальных сетей (`NSAllowsLocalNetworking`)

### 3. Сообщения об ошибках
- **Файл:** `lib/services/api_service.dart`
- **Изменения:**
  - Убраны упоминания о проблемах с мобильными данными
  - Добавлена обработка ошибок SSL/TLS сертификатов
  - Упрощены сообщения об ошибках сети

### 4. UI предупреждения
- **Файл:** `lib/screens/login_screen.dart`
- **Изменения:**
  - Убрано предупреждение о блокировке мобильных данных
  - Индикатор мобильного интернета теперь информационный (синий вместо оранжевого)
  - Текст: "Используется мобильный интернет. Приложение работает через HTTPS."

## Требования к серверу

⚠️ **ВАЖНО:** Сервер должен быть настроен для работы с HTTPS:

1. **SSL сертификат:**
   - Рекомендуется использовать валидный сертификат от доверенного CA (Let's Encrypt, Cloudflare и т.д.)
   - Если используется самоподписанный сертификат, iOS может требовать дополнительных настроек

2. **Порт:**
   - HTTPS обычно работает на порту 443
   - Если используется порт 3000, убедитесь, что он открыт для HTTPS трафика

3. **WebSocket (WSS):**
   - Socket.IO должен поддерживать WSS (WebSocket Secure)
   - URL должен быть `wss://` вместо `ws://`

## Проверка работы

1. **Через Wi-Fi:**
   - Приложение должно работать как обычно
   - Все запросы идут через HTTPS

2. **Через мобильные данные:**
   - ✅ Теперь должно работать через мобильный интернет Мегафона и других операторов
   - HTTPS трафик не блокируется операторами

3. **Проверка сертификата:**
   - Откройте `https://83.166.246.225:3000` в Safari
   - Если сертификат самоподписанный, Safari может показать предупреждение
   - В приложении это обрабатывается автоматически через настройки ATS

## Возможные проблемы

### 1. Ошибка SSL сертификата
**Симптомы:**
- Ошибка "Certificate" или "TLS" в логах
- Запросы не проходят

**Решение:**
- Убедитесь, что сервер использует валидный SSL сертификат
- Если используется самоподписанный сертификат, может потребоваться добавить его в доверенные в iOS

### 2. WebSocket не подключается
**Симптомы:**
- Сообщения не отправляются
- Ошибка подключения Socket.IO

**Решение:**
- Проверьте, что сервер поддерживает WSS (WebSocket Secure)
- Убедитесь, что порт открыт для WSS трафика

### 3. Таймауты подключения
**Симптомы:**
- Долгие задержки при запросах
- Таймауты

**Решение:**
- Проверьте скорость интернета
- Убедитесь, что сервер обрабатывает HTTPS запросы быстро

## Откат на HTTP (если нужно)

Если возникнут проблемы с HTTPS, можно временно вернуться на HTTP:

1. В `lib/config/api_config.dart` изменить:
   ```dart
   static const String baseUrl = 'http://83.166.246.225:3000';
   static const String wsUrl = 'http://83.166.246.225:3000';
   ```

2. В `ios/Runner/Info.plist` вернуть настройки для HTTP:
   ```xml
   <key>NSExceptionAllowsInsecureHTTPLoads</key>
   <true/>
   ```

⚠️ **Но помните:** HTTP не будет работать через мобильные данные Мегафона!
