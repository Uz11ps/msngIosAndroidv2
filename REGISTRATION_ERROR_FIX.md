# Исправление ошибки регистрации: FormatException

## Проблема

При попытке регистрации появляется ошибка:
```
Ошибка регистрации: FormatException: Unexpected character (at character 1)
```

И видно начало HTML кода (`<meta name="viewport"...`).

## Причина

Приложение Flutter ожидает JSON ответ от сервера, но получает HTML страницу. Это происходит когда:

1. **Сервер недоступен** - возвращается HTML страница ошибки (404, 500)
2. **Неверный URL эндпоинта** - сервер возвращает HTML страницу "Not Found"
3. **Сервер не запущен** - запрос не доходит до сервера, возвращается HTML ошибка
4. **Проблемы с сетью** - прокси или роутер возвращает HTML страницу ошибки

## Решение

Я исправил код в `lib/services/api_service.dart`:

1. ✅ Добавлена проверка Content-Type заголовка
2. ✅ Добавлена проверка формата ответа (должен начинаться с `{` или `[`)
3. ✅ Добавлена обработка ошибок с понятными сообщениями
4. ✅ Добавлено логирование для отладки

Теперь приложение:
- Проверяет формат ответа перед парсингом
- Показывает понятное сообщение об ошибке
- Логирует детали для отладки

## Что нужно проверить

### 1. Убедитесь, что сервер запущен

На сервере выполните:

```bash
cd бекенд
node index.js
```

Или проверьте, запущен ли процесс:

```bash
pm2 status
# или
ps aux | grep node
```

### 2. Проверьте доступность сервера

Откройте в браузере:
```
http://83.166.246.225:3000/api/auth/email-register
```

Должна вернуться ошибка JSON (не HTML):
```json
{"success":false,"message":"Email и пароль обязательны"}
```

Если возвращается HTML - сервер не запущен или неверный URL.

### 3. Проверьте настройки брандмауэра

Убедитесь, что порт 3000 открыт на сервере:

```bash
# На сервере
sudo ufw allow 3000
# или
sudo iptables -A INPUT -p tcp --dport 3000 -j ACCEPT
```

### 4. Проверьте логи сервера

Если сервер запущен через PM2:

```bash
pm2 logs messenger-backend
```

Ищите ошибки при запросах регистрации.

4. Проверьте логи в консоли - там будет подробная информация о запросе и ответе

## Дополнительные проверки

Если проблема остается:

1. **Проверьте URL в `lib/config/api_config.dart`:**
   ```dart
   static const String baseUrl = 'http://83.166.246.225:3000';
   ```

2. **Проверьте, что эндпоинт правильный:**
   ```dart
   static const String emailRegister = '/api/auth/email-register';
   ```

3. **Проверьте CORS на сервере** - должен быть настроен для вашего домена/IP

4. **Проверьте, что сервер слушает на правильном интерфейсе:**
   ```javascript
   // В index.js должно быть:
   server.listen(PORT, '0.0.0.0', () => {
     console.log(`Server running on port ${PORT}`);
   });
   ```

## Если сервер на другом адресе

Если сервер работает на другом адресе или порту, измените в `lib/config/api_config.dart`:

```dart
static const String baseUrl = 'http://ВАШ_IP:ВАШ_ПОРТ';
```
