# Решение проблем с TestFlight

## Проблема: У заказчика не работает авторизация, а у вас все работает

### Возможные причины:

1. **iOS App Transport Security (ATS)** - блокирует HTTP запросы
2. **Проблемы с сетью** - разные операторы/провайдеры
3. **Таймауты** - медленное соединение
4. **Кэш приложения** - старые данные
5. **Версия iOS** - разные версии могут вести себя по-разному

## Решения

### 1. Проверка настроек ATS

Убедитесь, что в `ios/Runner/Info.plist` есть:

```xml
<key>NSAppTransportSecurity</key>
<dict>
    <key>NSAllowsArbitraryLoads</key>
    <true/>
</dict>
```

### 2. Использование диагностического экрана

В приложении добавлен экран диагностики. Заказчик может:

1. Открыть экран входа
2. Нажать кнопку "Диагностика подключения" внизу
3. Увидеть детальную информацию о проблемах

### 3. Улучшенная обработка ошибок

Теперь приложение показывает более понятные сообщения:

- ✅ "Ошибка сети: ... Проверьте интернет-соединение"
- ✅ "Сервер вернул неверный формат данных. Возможно, сервер недоступен"
- ✅ "Превышено время ожидания. Проверьте интернет-соединение"

### 4. Что проверить у заказчика

1. **Интернет-соединение**
   - Работает ли интернет в других приложениях?
   - Может быть блокировка корпоративным WiFi/провайдером?

2. **Версия iOS**
   - Какая версия iOS на устройстве?
   - Старые версии могут иметь проблемы

3. **Логи приложения**
   - Попросите заказчика включить логирование (если возможно)
   - Или использовать диагностический экран

4. **Переустановка приложения**
   - Удалить приложение полностью
   - Переустановить из TestFlight

### 5. Диагностический экран

Экран диагностики проверяет:

- ✅ Доступность сервера
- ✅ Работу эндпоинта регистрации
- ✅ Работу эндпоинта входа
- ✅ Формат ответов от сервера

### 6. Что делать, если проблема остается

1. **Проверьте сервер**
   ```bash
   curl -X POST http://83.166.246.225:3000/api/auth/email-register \
     -H "Content-Type: application/json" \
     -d '{}'
   ```

2. **Проверьте логи сервера**
   ```bash
   pm2 logs messenger-backend
   ```

3. **Проверьте доступность с разных сетей**
   - Попросите заказчика попробовать с мобильного интернета
   - Попробовать с другого WiFi

4. **Создайте тестовый аккаунт**
   - Дайте заказчику тестовые данные для входа
   - Проверьте, работает ли регистрация

### 7. Альтернативное решение: HTTPS

Если проблема в ATS, можно настроить HTTPS:

1. Установить SSL сертификат на сервер
2. Изменить URL в `lib/config/api_config.dart` на `https://`
3. Обновить настройки сервера

Но это требует дополнительной настройки сервера.

## Инструкция для заказчика

Если у вас проблемы с авторизацией:

1. **Откройте приложение**
2. **На экране входа нажмите "Диагностика подключения"** (внизу экрана)
3. **Дождитесь завершения проверки**
4. **Сделайте скриншот результатов**
5. **Отправьте разработчику**

Также попробуйте:

- Переключиться на мобильный интернет (если используете WiFi)
- Перезапустить приложение
- Переустановить приложение из TestFlight
- Проверить, работает ли интернет в других приложениях

## Дополнительная диагностика

Если диагностический экран показывает ошибки:

1. **Ошибка "Сервер недоступен"**
   - Проверьте интернет-соединение
   - Попробуйте другую сеть
   - Проверьте, не блокирует ли провайдер доступ

2. **Ошибка "Сервер вернул не JSON"**
   - Возможно, сервер недоступен или перегружен
   - Свяжитесь с разработчиком

3. **Ошибка "Превышено время ожидания"**
   - Медленное интернет-соединение
   - Попробуйте другую сеть
   - Проверьте скорость интернета
