# Решение проблем: Почему у разработчика работает, а у клиента нет

## Возможные причины проблемы

### 1. Различия в сетевых настройках iOS

**Проблема:** iOS в Release сборке (TestFlight) может быть строже с HTTP трафиком, чем в Debug.

**Решение:** ✅ Уже исправлено
- Добавлены дополнительные настройки `NSAppTransportSecurity` в `Info.plist`
- Добавлен `NSAllowsLocalNetworking` для локальных сетей
- Добавлен `NSExceptionMinimumTLSVersion` для совместимости

### 2. Проблемы с мобильными данными (Cellular) ✅ РЕШЕНО

**Проблема:** Оператор **Мегафон** (и некоторые другие операторы) блокирует HTTP трафик через мобильные данные.

**Решение:** ✅ **ПЕРЕХОД НА HTTPS (версия 11+)**

Приложение переведено на HTTPS, что решает проблему с блокировкой HTTP трафика операторами.

**Что было сделано:**
- ✅ Переход с HTTP на HTTPS для всех API запросов
- ✅ Переход с WS на WSS для WebSocket соединений
- ✅ Обновлены настройки ATS для HTTPS
- ✅ Убраны предупреждения о мобильных данных (теперь не нужны)
- ✅ Обновлены сообщения об ошибках для HTTPS

**Текущее состояние:**
- ✅ Работает через Wi-Fi (как и раньше)
- ✅ **Теперь работает через мобильный интернет Мегафона и других операторов**
- ✅ HTTPS трафик не блокируется операторами

**Требования к серверу:**
- ⚠️ Сервер должен быть настроен для работы с HTTPS
- ⚠️ Должен быть установлен SSL сертификат (рекомендуется валидный от доверенного CA)
- ⚠️ WebSocket должен поддерживать WSS (WebSocket Secure)

**Подробности:** См. `HTTPS_MIGRATION.md`

### 3. Корпоративные/публичные Wi-Fi сети

**Проблема:** Многие публичные Wi-Fi (кафе, отели, аэропорты) блокируют HTTP трафик или требуют авторизации через браузер.

**Что проверить у клиента:**
- Попросите клиента попробовать с мобильным интернетом (отключить Wi-Fi)
- Попросите клиента открыть `http://83.166.246.225:3000` в Safari - если не открывается, значит сеть блокирует HTTP

**Решение:** Клиент должен использовать мобильный интернет или другую Wi-Fi сеть

### 4. VPN и прокси

**Проблема:** VPN или корпоративный прокси могут блокировать или изменять HTTP трафик.

**Что проверить:**
- Отключить VPN на устройстве клиента
- Проверить настройки прокси: Настройки → Wi-Fi → (i) рядом с сетью → Прокси-сервер

**Решение:** Отключить VPN/прокси перед тестированием

### 5. Различия в версиях iOS

**Проблема:** Новые версии iOS (iOS 17+) имеют более строгие правила безопасности.

**Что проверить:**
- Версия iOS у клиента (Настройки → Основные → Об этом устройстве)
- Минимальная версия в проекте: iOS 14.0

**Решение:** ✅ Уже настроено для iOS 14.0+

### 6. Кеширование и старые данные

**Проблема:** Старые данные или кеш могут мешать новым запросам.

**Решение для клиента:**
1. Удалить приложение полностью
2. Перезагрузить iPhone
3. Установить заново из TestFlight
4. Попробовать войти/зарегистрироваться

### 7. Отсутствие retry механизма

**Проблема:** Если первый запрос не прошел из-за временной проблемы сети, приложение сразу показывало ошибку.

**Решение:** ✅ Исправлено
- Добавлен автоматический retry механизм (3 попытки с задержкой)
- Улучшена обработка сетевых ошибок
- Добавлены детальные логи для диагностики

### 8. Проблемы с DNS

**Проблема:** У клиента могут быть проблемы с DNS, из-за чего IP адрес не резолвится.

**Что проверить:**
- Попросите клиента открыть `http://83.166.246.225:3000` в Safari
- Если не открывается - проблема с сетью или DNS

**Решение:** Использовать доменное имя вместо IP (если возможно) или проверить DNS настройки

### 9. Настройки приватности iOS

**Проблема:** В iOS есть настройки, которые могут блокировать сетевые запросы.

**Что проверить у клиента:**
- Настройки → Конфиденциальность → Безопасность
- Настройки → Конфиденциальность → Локальная сеть → Ваше приложение (должно быть включено)
- Настройки → Конфиденциальность → Отслеживание (не должно блокировать)

### 10. Ограничения родительского контроля

**Проблема:** Если на устройстве включен родительский контроль, он может блокировать сетевые запросы.

**Что проверить:**
- Настройки → Экранное время → Ограничения контента и конфиденциальности

## Что было исправлено в коде

### 1. Retry механизм с поддержкой мобильных сетей
- Для WiFi: до 3 попыток с задержкой 2s, 4s, 6s
- Для мобильных данных: до 5 попыток с задержкой 3s, 6s, 9s, 12s, 15s
- Экспоненциальная задержка с увеличенным множителем для мобильных данных
- Не повторяются только ошибки валидации (401, 403, FormatException)

### 2. Определение типа сети
- Автоматическое определение WiFi vs мобильные данные
- Специальная обработка для мобильных сетей
- Улучшенные сообщения об ошибках в зависимости от типа сети

### 3. Улучшенные заголовки
- Добавлен `User-Agent` для лучшей совместимости
- Добавлен `Connection: keep-alive` для переиспользования соединений

### 4. Улучшенные настройки ATS для мобильных сетей
- Добавлен `NSAllowsLocalNetworking` для локальных сетей
- Добавлен `NSAllowsArbitraryLoadsInWebContent` для веб-контента
- Добавлен `NSAllowsArbitraryLoadsForMedia` для медиа контента
- Добавлен `NSExceptionMinimumTLSVersion` для совместимости
- Добавлен `NSThirdPartyExceptionAllowsInsecureHTTPLoads` для сторонних сервисов
- Добавлен `NSThirdPartyExceptionRequiresForwardSecrecy: false` для HTTP трафика

### 5. Специальные сообщения об ошибках для мобильных данных
- Понятные сообщения о блокировке HTTP трафика операторами
- Рекомендации использовать Wi-Fi при проблемах с мобильными данными
- Детальная диагностика проблем с мобильными сетями

### 6. Детальное логирование
- Все запросы логируются с полной информацией
- Логируются попытки retry с указанием типа сети
- Логируется состояние сети при запуске
- Логируется тип сети (WiFi/Cellular) для каждого запроса

## Инструкция для клиента

Попросите клиента выполнить следующие шаги:

1. **Проверить интернет:**
   - Открыть Safari
   - Попробовать открыть любой сайт
   - Если не работает - проблема с интернетом

2. **Проверить работу приложения:**
   - ✅ Приложение теперь работает через Wi-Fi и мобильный интернет
   - ✅ Используется HTTPS, который не блокируется операторами
   - Если возникают проблемы, попробуйте переключиться между Wi-Fi и мобильным интернетом

3. **Если проблемы сохраняются:**
   - Проверьте версию приложения (должна быть 11+)
   - Переустановите приложение
   - Проверьте настройки сети на устройстве

3. **Отключить VPN/Прокси:**
   - Настройки → VPN (если есть - отключить)
   - Настройки → Wi-Fi → (i) → Прокси-сервер (должно быть "Выкл")

4. **Проверить доступность сервера:**
   - Открыть Safari
   - Ввести адрес: `http://83.166.246.225:3000`
   - Если не открывается - сеть блокирует HTTP трафик

5. **Переустановить приложение:**
   - Удалить приложение полностью
   - Перезагрузить iPhone
   - Установить заново из TestFlight
   - Попробовать войти/зарегистрироваться

6. **Собрать логи:**
   - Если проблема сохраняется, включить Share Analytics в TestFlight
   - Попробовать воспроизвести проблему
   - Отправить скриншоты ошибок разработчику

## Диагностика через логи

После включения детального логирования, в логах будут видны:

- 🔄 Попытки retry запросов
- ✅ Успешные запросы
- ❌ Ошибки с деталями
- 🌐 Состояние сети при запуске
- 📡 Полные ответы сервера

Ищите в логах:
- `🔄 Registration: Attempt X/3` - попытки повтора
- `❌ Registration failed:` - детали ошибки
- `💥 Network error:` - проблемы с сетью
- `⏱️ Registration request timeout` - таймауты

## Если проблема все еще сохраняется

Если после всех исправлений проблема сохраняется:

1. **Проверить логи сервера** - доходят ли запросы от клиента
2. **Проверить IP адрес** - возможно, у клиента другой IP или блокировка
3. **Рассмотреть использование HTTPS** - более надежно, но требует сертификат
4. **Добавить доменное имя** - вместо IP адреса использовать домен
