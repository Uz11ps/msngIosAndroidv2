# Автоматическое обновление бэкенда

## Описание

Скрипт `update_backend.py` автоматически:
1. ✅ Подключается к серверу по SSH
2. ✅ Загружает обновленные файлы (`index.js`, `package.json`)
3. ✅ Устанавливает зависимости через npm
4. ✅ Перезапускает сервер через PM2
5. ✅ Проверяет статус и доступность API
6. ✅ Показывает логи для отладки

## Использование

### Просто запустите скрипт:

```bash
python update_backend.py
```

или

```bash
python3 update_backend.py
```

### Что делает скрипт:

1. **Проверка файлов** - убеждается, что `бекенд/index.js` и `бекенд/package.json` существуют
2. **Подключение** - подключается к серверу `83.166.246.225` по SSH
3. **Загрузка** - загружает файлы в `/root/messenger/backend/`
4. **Установка** - запускает `npm install --production`
5. **Перезапуск** - перезапускает через PM2:
   - Если процесс уже запущен → `pm2 restart messenger-backend`
   - Если не запущен → `pm2 start index.js --name messenger-backend`
6. **Проверка** - проверяет статус PM2, логи и доступность API

## Требования

- Python 3
- Установленные библиотеки:
  ```bash
  pip install paramiko scp
  ```

## Настройка сервера

Скрипт использует следующие настройки:
- **Сервер**: `83.166.246.225`
- **Порт SSH**: `22`
- **Пользователь**: `root`
- **Путь на сервере**: `/root/messenger/backend/`
- **Имя процесса PM2**: `messenger-backend`

Если нужно изменить настройки, отредактируйте переменные в начале функции `deploy()`.

## Что проверяется после деплоя

1. ✅ Статус процесса в PM2
2. ✅ Последние 15 строк логов
3. ✅ Доступность API эндпоинта `/api/auth/email-register`

## Устранение проблем

### Ошибка подключения
- Проверьте, что сервер доступен
- Проверьте правильность IP адреса и пароля

### Ошибка при установке зависимостей
- Проверьте интернет-соединение на сервере
- Убедитесь, что npm установлен

### Сервер не запускается
- Проверьте логи: `pm2 logs messenger-backend`
- Убедитесь, что порт 3000 свободен
- Проверьте синтаксис `index.js`

### API не отвечает
- Подождите несколько секунд после перезапуска
- Проверьте логи на наличие ошибок
- Убедитесь, что база данных доступна
